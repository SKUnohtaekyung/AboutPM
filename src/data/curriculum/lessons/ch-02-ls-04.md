# Ch.2 — Lesson 4/7: 유저 저니 맵과 인지 부하 관리

> **학습 위치**: Chapter 2 · 4번째 레슨 / 전체 7개 레슨  
> **이전 단계**: 단편적인 고객 정보가 아닌, 시간의 흐름에 따른 고객의 경험을 추적합니다.  
> **다음 단계**: 설계된 경험이 비즈니스 성과로 이어지는지 데이터를 통해 확인합니다.

---

## 학습 목표
- 사용자 여정 지도를 그리고 페인포인트를 찾는다
- 안티패턴을 통해 좋은 UX의 기준을 정립한다

## 핵심 용어
- **유저 저니 맵 (User Journey Map)**
- **페인포인트 찾기와 가설 수립**

---

**0) 한 줄 정의**  
저니 맵은 “사용자가 목표를 이루는 동안 거치는 과정을 시간순으로 그린 지도”입니다.

![작전 지도 - 유저 저니 맵](/generated/map_room.svg) 

**1) 왜 필요한가**  
저니 맵은 ‘화면’이 아니라 ‘경험’을 다룹니다. 어디에서 감정이 떨어지고(좌절), 어디에서 가치가 생기며(만족), 어디가 병목인지(이탈)를 한눈에 보여줍니다. 

**2) 언제 쓰나**  
- “기능은 많은데 왜 안 쓰지?”가 반복될 때  
- 온보딩/결제/공유 같은 핵심 여정에서 이탈이 클 때  
- 여러 채널(웹/앱/메일/CS)이 얽혀 문제 원인이 분산될 때

**3) 어떻게 쓰나(단계 + 최소 산출물)**  
- 페르소나를 고른다(누구의 여정인가)  
- 단계 정의(인지→유입→탐색→행동→재방문 등)  
- 각 단계에서 행동/생각/감정을 정성 데이터로 채운다  
- Pain Point와 기회요인을 “가설”로 만든다  
- 최소 산출물: 저니 맵 1장 + 개선 가설 3개

**4) 예시 2개(가상)**  
- 예시A(B2C)  
상황: 유입은 많은데 가입 후 첫 과업 완료가 낮다.  
PM의 선택: 저니 맵에서 “가입 직후” 감정이 급락(정보 입력 피로)하는 것을 발견 → 입력을 1단계로 줄이고 나머지는 사용 중에 채우도록 변경.  
결과: 가입→첫 과업 완료 전환율↑, Activation↑.

- 예시B(B2B)  
상황: 사용자가 첫 대시보드를 만들기는 하지만 팀 공유를 안 한다.  
PM의 선택: “팀 공유” 단계에서 ‘권한 설정’이 불안(누가 볼까)하다는 감정 하락을 발견 → 기본 권한 정책/미리보기/감사로그 안내를 추가.  
결과: 공유 기능 사용률↑, 팀 단위 리텐션↑.

**5) 안티패턴**  
- ‘그림만 예쁜’ 저니 맵(리서치 근거가 없다)   
- 단계가 UI 화면 순서로만 구성되어, 감정/결정 포인트를 놓침  
- Pain Point를 “개선 아이디어”로 바로 점프(가설 검증이 없다)

**6) 체크리스트(복붙용)**  
- [ ] 저니 맵은 “목표 1개” 중심이다  
- [ ] 행동(Action)과 감정(Emotion)이 분리되어 있다  
- [ ] 최저점(골짜기) 1~2개가 명확하다  
- [ ] 골짜기마다 “원인 가설”이 있다

**7) 관련 지표/측정**  
- 단계별 전환율(퍼널), 과업 성공률(UT), 이탈률  
- 감정은 정량화 어려우므로 “대리지표”(예: 중간 이탈률, 문의 발생률)을 사용

| 단계 | 사용자 행동 | 사용자 감정(예) | 페인포인트(가설) | 기회(실험 아이디어) |
|---|---|---|---|---|
| 인지 | 앱스토어에서 발견 | 🙂 기대 | 가치가 불명확 | 첫 화면 문구/스크린샷 개선 |
| 가입 | 이메일/비번 입력 | 😐 귀찮음 | 입력이 많음 | 소셜 로그인/단계 축소 |
| 첫 사용 | 할 일 추가 | 🙂 괜찮음 | 우선순위가 어려움 | “오늘 3개 추천” |
| 재방문 | 다음날 다시 열기 | 😶 무관심 | 돌아올 이유 부족 | 리마인드/성과 피드백 |
| 습관화 | 주간 계획 | 🙂 만족 | 일정 변화에 취약 | 자동 재정렬/캘린더 연동 |

실무 체크리스트  
- [ ] 페르소나/저니 맵은 “예쁘게 만들기”가 아니라 “우선순위 기준”으로 쓴다   
- [ ] 정성(인터뷰/VOC)→정량(지표) 연결 고리를 PRD로 문서화한다   

자주 하는 실수(안티패턴)  
- 페르소나/저니 맵을 만든 뒤 backlog/PRD와 분리(아카이브화)  
- 리서치를 “기능 정답 맞히기”로 오해(문제/맥락을 놓침)
